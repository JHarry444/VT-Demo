# Exercise 4: Network Tab Instructions

In this exercise, you will learn how to use the Network tab in the Developer Tools to monitor and analyze network requests made by a JavaScript application. Follow the instructions below to complete the exercise.

## Objectives
- Understand how to open the Network tab in Developer Tools.
- Monitor network requests made by the application.
- Analyze the details of each request, including headers, response, and timing.

## Instructions

1. **Open the Application**:
   - Open the `index.html` file located in the `exercise-4-network` folder in your web browser.

2. **Access Developer Tools**:
   - Right-click anywhere on the page and select "Inspect" or press `F12` to open Developer Tools.

3. **Navigate to the Network Tab**:
   - Click on the "Network" tab in Developer Tools. This tab will show all network requests made by the application.

4. **Reload the Page**:
   - Refresh the page (press `F5` or click the refresh button in your browser). This will allow you to see all network requests that occur during the loading of the page.

5. **Analyze Network Requests**:
   - Observe the list of network requests that appear in the Network tab. You should see requests for various resources such as HTML, CSS, JavaScript, and any API calls made by the application.
   - Click on any request to view its details. Pay attention to the following:
     - **Headers**: Check the request and response headers to understand what data is being sent and received.
     - **Response**: View the response data returned by the server.
     - **Timing**: Analyze the timing information to see how long each request took to complete.

6. **Filter Requests**:
   - Use the filter options to narrow down the requests by type (e.g., XHR, JS, CSS) to focus on specific types of network activity.

7. **Experiment with the Application**:
   - Interact with the application (e.g., click buttons, submit forms) to trigger additional network requests. Observe how these requests appear in the Network tab.

8. **Take Notes**:
   - Document any interesting findings or issues you encounter while monitoring the network requests. Consider how this information could help in debugging or optimizing the application.

## Conclusion
By the end of this exercise, you should have a better understanding of how to use the Network tab in Developer Tools to monitor and analyze network activity in a JavaScript application. This skill is essential for debugging issues related to data fetching and performance optimization.
